数据迁移属性

本文最后更新于：2021-08-15 18:08

flyway

配置项默认值说明
spring.flyway.baseline-description<< Flyway Baseline >>在应用基线时，对现有模式进行标记的描述。
spring.flyway.baseline-on-migratefalse迁移非空模式时是否自动调用基线。
spring.flyway.baseline-version1在执行基线时，对现有模式进行标记的版本。
spring.flyway.batch执行 SQL 语句时，是否要批处理。需要 Flyway Pro 或 Flyway Enterprise。
spring.flyway.check-locationtrue是否检查迁移脚本的位置是否存在。
spring.flyway.clean-disabledfalse是否禁止清理数据库。
spring.flyway.clean-on-validation-errorfalse当发生验证错误时，是否自动调用 clean。
spring.flyway.connect-retries0.0试图连接到数据库时的最大重试次数。
spring.flyway.default-schema由 Flyway 管理的默认模式名称（区分大小写）。
spring.flyway.enabledtrue是否启用飞道。
spring.flyway.encodingUTF-8SQL 迁移的编码。
spring.flyway.error-overrides内置的错误处理规则，用于覆盖特定的 SQL 状态和错误代码。需要 Flyway Pro 或 Flyway Enterprise。
spring.flyway.groupfalse在申请迁移时，是否将所有待处理的迁移集中在同一个事务中。
spring.flyway.ignore-future-migrationstrue读取模式历史表时是否忽略未来的迁移。
spring.flyway.ignore-ignored-migrationsfalse读取模式历史表时是否忽略被忽略的迁移。
spring.flyway.ignore-missing-migrationsfalse读取模式历史表时是否忽略缺失的迁移。
spring.flyway.ignore-pending-migrationsfalse读取模式历史表时是否忽略待迁移。
spring.flyway.init-sqls获得连接后，立即执行 SQL 语句来初始化连接。
spring.flyway.installed-by在模式历史表中记录的应用迁移的用户名。
spring.flyway.license-keyFlyway Pro 或 Flyway Enterprise 的许可证密钥。
spring.flyway.locations[classpath:db/migration]迁移脚本的位置。可以包含特殊的”{vendor}”占位符，以使用特定供应商的位置。
spring.flyway.mixedfalse是否允许在同一迁移中混合事务性和非事务性语句。
spring.flyway.oracle-sqlplus是否启用对 Oracle SQL*Plus 命令的支持。需要 Flyway Pro 或 Flyway Enterprise。
spring.flyway.oracle-sqlplus-warn当遇到尚未支持的 Oracle SQL*Plus 语句时，是否要发出警告而不是错误。需要 Flyway Pro 或 Flyway Enterprise。
spring.flyway.out-of-orderfalse是否允许迁移运行不正常。
spring.flyway.password要迁移的数据库的登录密码。
spring.flyway.placeholder-prefix${迁移脚本中占位符的前缀。
spring.flyway.placeholder-replacementtrue在迁移脚本中执行占位符替换。
spring.flyway.placeholder-suffix}迁移脚本中占位符的后缀。
spring.flyway.placeholders.*占位符和它们的替换适用于 sql 迁移脚本。
spring.flyway.repeatable-sql-migration-prefixR可重复的 SQL 迁移的文件名前缀。
spring.flyway.schemas由 Flyway 管理的计划名称（区分大小写）。
spring.flyway.skip-default-callbacksfalse是否跳过默认回调。如果为 true，则只使用自定义回调。
spring.flyway.skip-default-resolversfalse是否跳过默认的解析器。如果为真，则只使用自定义解析器。
spring.flyway.sql-migration-prefixVSQL 迁移的文件名前缀。
spring.flyway.sql-migration-separator__SQL 迁移的文件名分隔符。
spring.flyway.sql-migration-suffixes[.sql]SQL 迁移的文件名后缀。
spring.flyway.stream执行 SQL 迁移时是否要流式迁移。需要 Flyway Pro 或 Flyway Enterprise。
spring.flyway.tableflyway_schema_historyFlyway 将使用的模式历史表的名称。
spring.flyway.tablespace创建模式历史表的表空间。当使用不支持表空间的数据库时，忽略。默认为 Flyway 使用的连接的默认表空间。
spring.flyway.target应考虑向其迁移的目标版本。
spring.flyway.url要迁移的数据库的 JDBC url，如果没有设置，则使用主要配置的数据源。如果没有设置，则使用主要配置的数据源。
spring.flyway.user要迁移的数据库的登录用户。
spring.flyway.validate-migration-namingfalse是否验证那些脚本不遵守正确命名惯例的迁移和回调。
spring.flyway.validate-on-migratetrue是否在进行迁移时自动调用验证。

liquibase

配置项默认值说明
spring.liquibase.change-logclasspath:/db/changelog/db.changelog-master.yaml更改日志配置路径。
spring.liquibase.clear-checksumsfalse是否清除当前变更日志中的所有校验和，以便在下次更新时重新计算。
spring.liquibase.contexts以逗号分隔的运行时上下文列表。
spring.liquibase.database-change-log-lock-tableDATABASECHANGELOGLOCK用来跟踪同时使用 Liquibase 的表的名称。
spring.liquibase.database-change-log-tableDATABASECHANGELOG用于跟踪变更历史的表的名称。
spring.liquibase.default-schema默认的数据库模式。
spring.liquibase.drop-firstfalse是否先放弃数据库模式。
spring.liquibase.enabledtrue是否启用 Liquibase 支持。
spring.liquibase.labels用逗号分隔的运行时标签列表。
spring.liquibase.liquibase-schema用于 Liquibase 对象的模式。
spring.liquibase.liquibase-tablespace用于 Liquibase 对象的表空间。
spring.liquibase.parameters.*更改日志参数。
spring.liquibase.password要迁移的数据库的登录密码。
spring.liquibase.rollback-file执行更新时，回滚 SQL 被写入的文件。
spring.liquibase.tag应用数据库更改时要使用的标记名称，也可以与 “rollbackFile “一起使用，为与该标记相关的所有现有更改生成回滚脚本。也可与 “rollbackFile “一起使用，为与该标记相关联的所有现有更改生成回滚脚本。
spring.liquibase.test-rollback-on-updatefalse更新前是否要进行回滚测试。
spring.liquibase.url要迁移的数据库的 JDBC URL，如果没有设置，则使用主要配置的数据源。如果没有设置，则使用主要配置的数据源。
spring.liquibase.user要迁移的数据库的登录用户。
