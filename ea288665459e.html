Hexo搭建教程(二)-熟悉官方文件、配置、命令

发表于2021-08-26|更新于2021-08-26|Hexo
|字数总计:2.8k|阅读时长:9分钟|阅读量:

前言：承接上文，现在已经成功搭建了一个不错的环境，但是我们对生成出来的文件还是感觉有点莫名其妙，而且不知道怎么去创建文章，怎么去修改一些配置，这次我们就来熟悉下官方提供的配置有哪些，但是应该不会出现截图，因为我的已经搭建完成，而且偷懒不想再搭建，具体呈现的效果需要自己去测试，好的Go~ Go~ Go~（当然这一章的内容很大部分都是官网，我只是抽出一部分需要注意的东西，而且和后面会出现一定的联动，请自己斟酌）

官方文件、配置、命令

以下的内容只是针对Windows

附上官网：https://hexo.io/zh-cn/docs/

文件夹及文件

成功生成文件后在选中的文件夹下面会生成一些文件，现在就来看看这些文件有上面作用

我们先看文件夹再看文件

themes（主题）

这个主题文件夹在hexo 5.0版本以后就没有太大作用了，一般都是改用npm i hexo-theme-主题名称来下载主题，就算需要自己去魔改主题都是推荐在Github找到对应主题，然后Fork到自己的仓库，然后做更改，这样在主仓库更新新功能的时候也可以及时更新和保留自己修改的文件

如果是现在安装Hexo版本都比这高了，所以可以不用关心这个文件夹了

scaffolds（模板）

这个是模板的文件夹里面只有三个文件，这三个主文件都是在使用命令hexo new post "名称" 和

hexo new page "名称" 才有用，按道理还有一个但是不常用就不说了，这名称对照一下应该就看出什么了，就是hexo new 文件名 名称，下面就是描述一下这三个文件的作用（只讲两个）

  • page.md：这个文件是管目录的，在使用hexo new page "名称"的时候会在source文件夹下生成一个文件夹和文件，而且名称和你输入的名称一样，而文件内容是

  • post.md：这个管文章的，在使用hexo new post "名称"的时候会在source/_posts文件夹下生成文件，而且名称和你输入的名称一样，而文件内容是

这两个文件主要是配合主题使用，因为不同的主题在里面添加的内容也不一样，但post.md提供了一部分是官方指定的参数，可以根据自己的情况添加

参数 描述 默认值 是否常用
layout 布局 config.default_layout 否
title 标题 文章的文件名 是
date 建立日期 文件建立日期 是
updated 更新日期 文件更新日期 是
comments 开启文章的评论功能 true 是
tags 标签（不适用于分页），支持数组 是
categories 分类（不适用于分页） 是
permalink 覆盖文章网址 否
excerpt 纯文本页面摘录。 使用这个插件格式化文本 否
disableNunjucks 启用时禁用 Nunjucks 标签{{ }}/{% %}和标签插件的呈现 否
lang 设置语言覆盖自动检测 Inherited from _config.yml 否

source（资源）

差不多全部需要编译的文件都在这个文件里面，什么目录、文章、图片…都在这个文件夹里面，官方提供了有文章（_posts）和草稿（_drafts），但是一般没有用草稿，一般都是在文章编写完成后才做上传，没写完不做提交就好了，如果有需要的也可以自己设置，这里面的内容都会通过编译后放在public文件夹下，自己添加的html页面也会所以在_config.yml会设置排除，这个不用担心

_config.yml（配置）

这里面的一大坨东西分批来看

Site（网站）

参数 描述 使用描述
title 网站标题 大标题
subtitle 网站副标题 小标题
description 网站描述 这个网站是做什么的主要是给SEO看
keywords 网站的关键词。支持多个关键词。 这个看心情填写
author 您的名字 网站的站长嘛，取个漂亮的名字没问题
language 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。 这个根据你自己观看情况填写吧
timezone 网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。一般的，对于中国大陆地区可以使用 Asia/Shanghai。 这个也是差不多看你心情

参考

URL（网址）

参数 描述 默认值 使用描述
url 网址, 必须以 http:// 或 https:// 开头 这个看情况更改，使用github还是coding或者自己的域名，都行
root 网站根目录 url's pathname 是否未子目录
permalink 文章的永久链接格式 :year/:month/:day/:title/ 这个默认的太长了，一般都会改，怎么改，下面有个描述，自己搭配
permalink_defaults 永久链接中各部分的默认值 对我没用
pretty_urls 改写 permalink 的值来美化 URL
pretty_urls.trailing_index 是否在永久链接中保留尾部的 index.html，设置为 false 时去除 true 没改
pretty_urls.trailing_html 是否在永久链接中保留尾部的 .html, 设置为 false 时去除 (对尾部的 index.html无效) true 没改

参考

网站存放在子目录

如果您的网站存放在子目录中，例如 http://example.com/blog，则请将您的 url 设为 http://example.com/blog 并把 root 设为 /blog/。

permalink （永久链接）
变量 描述
:year 文章的发表年份（4 位数）
:month 文章的发表月份（2 位数）
:i_month 文章的发表月份（去掉开头的零）
:day 文章的发表日期 (2 位数)
:i_day 文章的发表日期（去掉开头的零）
:hour 文章发表时的小时 (2 位数)
:minute 文章发表时的分钟 (2 位数)
:second 文章发表时的秒钟 (2 位数)
:title 文件名称 (relative to “source/_posts/“ folder)
:name 文件名称
:post_title 文章标题
:id 文章 ID (not persistent across cache reset)
:category 分类。如果文章没有分类，则是 default_category 配置信息。
:hash SHA1 hash of filename (same as :title) and date (12-hexadecimal)

我为了方便直接使用的是:hash.html，除了在设置默认方面会有点麻烦其他还好

Directory（目录）

没怎么改，就加了一个，看参考吧
参数 描述 默认值
source_dir 资源文件夹，这个文件夹用来存放内容。 source
public_dir 公共文件夹，这个文件夹用于存放生成的站点文件。 public
tag_dir 标签文件夹 tags
archive_dir 归档文件夹 archives
category_dir 分类文件夹 categories
code_dir Include code 文件夹，source_dir 下的子目录 downloads/code
i18n_dir 国际化（i18n）文件夹 :lang
skip_render 跳过指定文件的渲染。匹配到的文件将会被不做改动地复制到 public 目录中。您可使用 glob 表达式来匹配路径。

参考

Writing（文章）
参数 描述 默认值
new_post_name 新文章的文件名称 :title.md
default_layout 预设布局 post
auto_spacing 在中文和英文之间加入空格 false
titlecase 把标题转换为 title case false
external_link 在新标签中打开链接 true
external_link.enable 在新标签中打开链接 true
external_link.field 对整个网站（site）生效或仅对文章（post）生效 site
external_link.exclude 需要排除的域名。主域名和子域名如 www 需分别配置 []
filename_case 把文件名称转换为 (1) 小写或 (2) 大写 0
render_drafts 显示草稿 false
post_asset_folder 启动 Asset 文件夹 false
relative_link 把链接改为与根目录的相对位址 false
future 显示未来的文章 true
highlight 代码块的设置, 请参考 Highlight.js 进行设置
prismjs 代码块的设置, 请参考 PrismJS 进行设置

参考

杂项

下面的基本都没有做修改，可以直接看一下

  • theme：为主题，这个最开始的肯定不是这个，后面会做修改，也可以更具自己喜欢去找主题
  • deploy：这个是提交，但是后面不会在本地提交，就只用来提交百度的搜录，刚开始应该为空，这个也可以不用管

package.json（项目文件，没必要不要手动）

命令

这下面只使用去描述常用的命令，特别的命令请去官网查

常用命令

命令 作用
hexo new page '名称' 创建文件夹及文件（一般用来目录）
hexo new post '名称' 创建文章
hexo generate or hexo g 生成静态文件到public下
hexo server or hexo s 启动服务器。默认情况下，访问网址为： http://localhost:4000/。
hexo deploy or hexo d 部署网站
hexo clean 清除缓存
